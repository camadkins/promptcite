## Chain-of-custody policy

- Adding a maintainer requires:
  1. CLA signature recorded by the bot (no exceptions — applies to founders too)
  2. Pull request adding the maintainer to this file, signed off by all current maintainers
  3. GitHub team membership granted on the same PR merge
- Removing a maintainer requires:
  1. A PR moving the entry to "Former maintainers" with the departure date
  2. Optional rationale in the PR description
  3. GitHub team membership revoked on the same PR merge
- Transferring project ownership requires:
  1. All conditions above
  2. The outgoing maintainer's signed acknowledgment that the new maintainer inherits the right to administer the CLA program and dual-licensing decisions
  3. Update to `.cla-assistant.json`, `LICENSE-COMMERCIAL` contact, and `SECURITY.md` reporting address in the same PR

## Why this matters

PromptCite is dual-licensed (AGPL v3 + commercial). The commercial
license requires a clean, auditable chain of who could legally bind the
project to a relicensing decision at any given time. This file is that
chain. It is updated *in the same PR* as any role change so the git
history is the audit trail.

See `CLA.md` for the contributor agreement, `LICENSE-COMMERCIAL` for the
commercial program, and `NOTICE.md` for the plain-English explanation of
the dual-license model.
